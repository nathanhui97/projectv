# Schema Gaps — Cards Requiring New Schema or Encoding Work

Generated by auditing ST01–ST09, GD01–GD04 ability seed scripts.

## Legend — What's Missing

| Tag | Meaning |
|---|---|
| `[CONDITION]` | Needs a `ConditionSchema` — "if X then do Y" logic |
| `[CONTINUOUS]` | Needs `apply_continuous` step — ongoing buff/debuff while paired/linked |
| `[TAUNT]` | Needs `grant_taunt` step — force enemies to attack this unit |
| `[REDIRECT]` | Needs `change_attack_target` step — redirect enemy attack mid-battle |
| `[PREVENT]` | Needs `prevent_ready` or `prevent_action` step |
| `[MILL]` | Needs `mill` step — send top N cards from deck to trash |
| `[EXILE]` | Needs `exile` step — remove cards from game (usually from trash as cost) |
| `[SEARCH_TRASH]` | Needs `search_trash` step — look through trash, add card to hand |
| `[MOVE_TO_DECK]` | Needs `move_to_deck` step — return card to deck top/bottom/shuffle |
| `[DISCARD_HAND]` | Needs `discard_from_hand` step — player chooses a card from hand to discard |
| `[COST_REDUCE]` | Needs `modify_cost` — static hand cost reduction based on board state |
| `[COUNT_ZONE]` | Needs `count_zone` step — count matching cards in a zone, store as number |
| `[SCALE_EFFECT]` | Needs scaling damage/stat by a counted value |
| `[DEPLOY_TRASH]` | Needs `deploy_from_trash` — deploy a card directly from trash |
| `[EX_RESOURCE]` | Needs `add_ex_resource` step — place an EX Resource |
| `[DUAL_TARGET]` | Needs targeting two different groups in a single step |
| `[BATTLE_SCOPE]` | Needs effects scoped to the current battle window |
| `[DAMAGE_IMMUNITY]` | Needs `grant_immunity` — prevent damage from specific sources |
| `[NESTED_TRIGGER]` | Trigger-within-trigger (e.g. during_pair + when destroy → effect) |
| `[FILTER_NAME]` | Needs `name_includes` filter field |
| `[FILTER_KEYWORD]` | Needs `has_keyword` filter field |
| `[FILTER_ZONE]` | Needs `zone` filter field |
| `[SELF_STAT_SCALE]` | AP/HP that dynamically scales based on zone count |

## Recommended Phase Order

### Phase 1 — Schema Foundation (blocks everything else)
1. Write `ConditionSchema` with types: `count`, `board_count`, `self_has_stat`, `self_has_keyword`, `is_my_turn`, `pilot_has_trait`, `and` / `or` / `not`
2. Add `condition?: Condition` field to all step action types
3. Add `[CONTINUOUS]` encoding: `apply_continuous` step with duration `"while_paired"` / `"while_linked"`
4. Add missing filter fields: `has_keyword`, `name_includes`, `zone`
5. Add `[NESTED_TRIGGER]` pattern: separate top-level ability with the correct event trigger + qualifier

### Phase 2 — New Step Actions
6. `mill`, `move_to_deck`, `search_trash`, `exile`, `discard_from_hand`
7. `grant_taunt`, `change_attack_target`, `prevent_ready`
8. `count_zone` (store count as variable for later use)

### Phase 3 — Complex Patterns
9. `[COST_REDUCE]` / `[SELF_STAT_SCALE]` — static zone-based effects
10. `[BATTLE_SCOPE]` — timed effect windows
11. `[DAMAGE_IMMUNITY]` — `grant_immunity` step
12. `[SCALE_EFFECT]` — use `count_zone` result as amount
